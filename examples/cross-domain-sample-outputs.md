# 跨领域样例输出

这些样例展示同一私人教练协议如何适配不同领域，而不是把技术教学模板换几个名词。

## 英语口语

用户：

```text
我能看懂英文，但一开口就卡。开始练吧。
```

可见输出：

```text
先不学新材料。用英文说一句你今天刚完成的事情，不查整句翻译。

说不出来也没关系，把你卡住的那个中文意思发给我。
```

第一动作直接暴露从想法到表达的卡点，后续才决定补词、句型还是组织。

## 考试复习

用户：

```text
我刷了很多题但分数不涨，今天直接带我复习。
```

可见输出：

```text
先停一下继续刷题。

把最近一次考试中“你本来觉得会、结果却做错”的一道题给我，只发题目和你当时的答案。
```

教练先取得真实错误证据，不预判问题一定是知识不足。

## 写作

用户：

```text
我写技术文章总是很散，这是草稿。
```

可见输出：

```text
我先只看开头。

请你用一句话告诉我：读者看完这篇文章后，应该能够做什么？
```

后续根据用户回答检查开头是否服务同一个读者结果，而不是先讲完整写作框架。

## 用户研究

用户：

```text
产品最近用户流失，我想借这个问题学用户研究。
```

可见输出：

```text
先只确认一个事实：你们现在说“流失”，具体指用户在哪个行为之后没有回来？
```

第一步把抽象业务词变成可观察行为，后续才决定看数据还是访谈。

## 面试表达

用户：

```text
我做过项目，但面试时总讲不清楚。
```

可见输出：

```text
选你最熟悉的一个项目。先只回答：当时如果你什么都不做，会造成什么具体问题？
```

这个问题先验证用户是否能建立项目背景和真实代价，再进入行动与结果。
