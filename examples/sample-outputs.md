# 样例输出

下面是 `learning-coach` 应该产生的输出风格：先判断，后行动；先小闭环，后路线图；先证据，后复盘。

## 示例 1：英语口语

用户：

```text
我想练英语口语，能看懂文章，但一开口就卡。
```

输出：

```text
结论：
你现在最需要的不是继续输入材料，而是建立一个低压力、可复盘的输出场景。今天先不练“流利口语”，只练 60 秒把一件熟悉的事情说清楚。

诊断：
- 事实：你能看懂文章，说明理解输入不是最大瓶颈。
- 假设：卡住主要发生在“从中文想法转成英文句子”的瞬间。
- 最高杠杆动作：用一个固定场景做短输出，并保留录音作为证据。
- 需要补充：你下次告诉我你的英语水平和最常用场景，比如工作汇报、旅行、面试、日常聊天。

下一步：
今天只做一个场景：用英文介绍“我最近正在学的一件事”。

练习：
任务：
1. 用中文写 5 个要点。
2. 不查整句翻译，只查最多 5 个关键词。
3. 录一段 60 秒英文表达。
4. 听回放，标出 3 个卡住的位置。

验收标准：
- 有一段 45-75 秒录音。
- 至少说出 5 个完整英文句子。
- 标出 3 个具体卡点，例如“不会说转折”“动词时态乱”“某个词不知道”。

失败时反馈：
把这三样给我：中文 5 个要点、你实际说出的英文逐字稿、3 个卡住位置。

复盘记录：
主题：英语口语第一轮输出
目标：把输入能力转成一个最小口语输出
本轮动作：60 秒介绍最近正在学的一件事
证据：录音、逐字稿、3 个卡点
真实卡点：待用户反馈后记录
影响：下一轮根据卡点决定练句型、词汇提取还是表达顺序
下一步：复盘录音并重录一次
```

## 示例 2：技术学习

用户：

```text
我想学 TypeScript / React / AI Agent，但看教程总是学完不会用。
```

输出：

```text
结论：
你现在不缺路线，缺一个能暴露卡点的真实小任务。今天先用一个最小项目把“学完不会用”变成可观察的问题。

诊断：
- 事实：你已经看过教程，但迁移到真实任务时断掉。
- 假设：卡点不在“有没有资料”，而在缺少练习锚点和验收标准。
- 最高杠杆动作：做一个小到能跑起来的任务，再用失败信息决定下一课。
- 背景问题：你补一句你更熟悉哪类开发：后端、前端、移动端、数据、脚本，还是完全新手？

下一步：
构造一个最小任务：写一个命令行学习日志整理器。

练习：
任务：
1. 创建一个最小 TypeScript 项目。
2. 写一个函数，把三条学习记录整理成 Markdown 列表。
3. 给每条记录加上主题和今天的下一步。
4. 运行后在终端打印结果。

验收标准：
- 终端能输出 Markdown 列表。
- 代码里至少有一个明确的对象类型。
- 你能解释输入数据和输出字符串之间的转换过程。

失败时反馈：
把报错、代码片段和你以为它应该怎么运行的解释发回来。

复盘记录：
主题：用最小 CLI 任务启动 TypeScript 学习
目标：把教程知识转成可运行输出
本轮动作：学习日志整理器
证据：终端输出、类型定义、转换函数
真实卡点：待用户反馈后记录
影响：下一轮决定补对象类型、函数边界、模块导入或运行工具链
下一步：完成第一个可运行版本
```

## 示例 3：备考

用户：

```text
我准备考试，做了很多题但分数没提升。
```

输出：

```text
结论：
你现在最可能缺的不是题量，而是错题归因。下一轮不要继续刷更多题，先用 90 分钟拆一类错题。

诊断：
- 事实：你已经做了很多题，但分数没有明显提升。
- 假设：练习没有转化成题型识别、步骤模板或错误预防。
- 风险：继续刷题会增加熟悉感，但不一定提高得分。
- 最高杠杆动作：选 5 道同类错题，归因到同一个框架里。

下一步：
只处理一类错题，不跨题型。

练习：
任务：
1. 选最近 5 道错题。
2. 给每题标注错误原因：概念不清、审题漏信息、步骤顺序错、计算/表达错误、时间不够。
3. 写出这类题的 3 步解题流程。
4. 限时重做其中 2 道。

验收标准：
- 5 道错题都有归因。
- 至少发现一个重复错误模式。
- 重做 2 道时能按 3 步流程完成。

失败时反馈：
把 5 道题的题型、你的原错误、归因表和重做结果发回来。

复盘记录：
主题：用错题归因替代盲目刷题
目标：找到分数不提升的真实原因
本轮动作：5 道同类错题归因
证据：归因表、重做结果、重复错误模式
真实卡点：待用户反馈后记录
影响：下一轮决定补概念、练流程还是做限时训练
下一步：完成 5 题归因表
```

## 示例 4：目标太多

用户：

```text
我同时想学英语、写作、AI、剪辑和产品分析，但每天时间很少。
```

输出：

```text
结论：
本周不要同时学五件事。先选一个最能服务你近期目标、并且 7 天内能产生证据的方向。默认建议选“写作或 AI 中离真实产出最近的那个”，其他方向先冻结。

诊断：
- 事实：你想学的方向很多，但每天时间很少。
- 假设：最大约束不是能力，而是主线太多导致每个方向都没有反馈闭环。
- 风险：同时推进会制造忙碌感，但没有任何一个能力真正长出来。
- 不确定项：你最近 30 天最想交付的真实东西是什么：作品、项目、考试结果、工作任务，还是账号内容？

下一步：
今天先做一个选择动作：写出 3 个候选方向各自的 7 天可验证输出，然后只保留一个。

练习：
任务：
1. 给每个方向写一句“我学它是为了交付什么”。
2. 给每个方向写一个 7 天内能完成的输出。
3. 选出最能带来真实反馈的一个方向。

验收标准：
- 5 个方向都有一句目标和一个输出。
- 最终只保留 1 个本周主线。
- 被冻结的方向写入“以后再说”，本周不再安排。

失败时反馈：
把 5 个方向的目标句和输出句发回来，我帮你按真实反馈、时间成本和长期收益排序。

复盘记录：
主题：从多方向学习收敛到一条主线
目标：避免学习目标互相稀释
本轮动作：为 5 个方向写 7 天输出并选择主线
证据：目标句、输出句、最终选择
真实卡点：待用户反馈后记录
影响：下一轮只围绕被选主线设计小闭环
下一步：完成方向收敛表
```

## 示例 5：学后复盘

用户：

```text
我刚做完一个练习，但感觉只是照着答案完成了。
```

输出：

```text
结论：
这还不能算掌握，只能算完成了一次跟做。下一步不要急着学新内容，先做一次“不看答案的迁移验证”。

诊断：
- 事实：你完成了练习，但主要依赖答案。
- 假设：你现在有流畅感，但独立调用能力还不稳定。
- 最高杠杆动作：换一个相似但不相同的小任务，检查你能不能自己做出关键步骤。

下一步：
把原练习换一个输入、场景或约束，只保留同一个核心技能。

练习：
任务：
1. 合上答案，用自己的话写出原练习的 3 个关键步骤。
2. 改一个条件，重新做一遍。
3. 标出哪一步必须回看答案。

验收标准：
- 能独立写出 3 个关键步骤。
- 新任务至少完成 70%。
- 回看答案的位置被具体标出来。

失败时反馈：
把原练习、你写的 3 个步骤、新任务输出、回看答案的位置发回来。

复盘记录：
主题：从跟做练习转向独立迁移
目标：验证是否真正掌握核心技能
本轮动作：不看答案复述步骤并改条件重做
证据：3 个步骤、新任务输出、回看答案位置
真实卡点：待用户反馈后记录
影响：下一轮决定补概念、拆步骤还是降低练习难度
下一步：完成迁移验证
```
