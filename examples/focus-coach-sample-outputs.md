# focus-coach 样例输出

## 示例 1：多目标收敛

用户：

```text
我是 Java 开发，目标是进入 AI Agent 开发，手上有一个 TypeScript 的 mini-claude 项目。
我同时想学 TS、算法和多 Agent，但这个月只能重点投入一个。
```

输出：

```text
这个月主线选 TypeScript，真实项目用 mini-claude。算法只做最低维护，多 Agent 冻结到下一次阶段复盘。

依据：你当前目标是进入 Agent 开发，而读懂和修改 TypeScript Agent 是离真实输出最近的约束。现在直接学多 Agent 会同时叠加语言和架构负担。

本周期承诺：用四周完成一条真实 Agent 调用链的阅读、Debug 和一次低风险修改。证据不是“上了多少课”，而是你能独立指出入口、模型调用、工具执行和消息回填，并完成验证。

四周后复盘；如果 TypeScript 基础不再是主要约束，再决定是否恢复多 Agent。

方向已清楚，具体第一步交给 learning-coach 读取学习现场后决定。
```

## 示例 2：路线可能错误

用户：

```text
我学了两个月还是不会独立做项目，这条路线是不是错了？
```

输出：

```text
当前最大约束更像是缺少真实项目中的独立输出，而不是缺少下一门课程。这是基于你“持续学习但不能脱离材料完成任务”的描述做出的假设。

先暂停新增课程一周，只做一个验证：选择当前项目中的一个小功能，不看教程完成定位、预测、实现或解释，并保留报错和卡点。

一周后的证据决定路线：如果主要卡在基础知识，就针对性补；如果能完成但一直没有真实任务，保留方向、改变训练方式；如果项目与现实目标无关，再考虑转向。
```
