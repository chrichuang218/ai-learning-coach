# learning-coach 跨领域冒烟验证

这些用例验证“私人教练协议”不是技术专用模板。不同领域应使用不同真实锚点和反馈机制，但仍遵循主动备课、一次一步和证据掌握。

## 1. 英语口语

```text
我能看懂英文，但一开口就卡。开始练吧，你来带。
```

验收重点：

- 不先给完整口语课程、资源清单或长问卷。
- 选择一个真实表达场景，只给第一个表达动作。
- 反馈证据应来自实际表达、录音、转写或卡住句子。
- 后续根据用户真实输出决定补词汇、句型还是组织顺序。

## 2. 考试复习

```text
我刷了很多题但分数不涨，今天直接带我复习。
```

验收重点：

- 不直接安排更多题量或完整复习计划。
- 先从一份真实错题、最近试卷或得分证据进入。
- 第一轮只要求用户定位一个代表错误或提供一份证据。
- 根据错因决定知识、识别、步骤、速度或表达问题。

## 3. 写作

```text
我写技术文章总是很散。这里有一篇正在写的草稿，带我改。
```

验收重点：

- 先读取真实草稿，而不是讲通用写作理论。
- 选择标题、开头、一个段落或一条论证作为当前切片。
- 使用读者理解、结构变化或修改前后对比作为证据。

## 4. 用户研究

```text
我想学用户研究，手上正好有一个用户流失问题，不知道先看什么。
```

验收重点：

- 以真实业务问题和现有证据为锚点。
- 第一轮只推进一个研究动作，例如澄清一个假设或检查一类证据。
- 不生成完整研究课程或一次性研究方案。
- 后续反馈来自访谈、行为数据或假设是否被证据支持。

## 5. 论文阅读

```text
这篇论文我读不进去，你从哪里开始由你决定。
```

验收重点：

- 先检查论文、摘要、图表或用户目标。
- 选择一个能打开全文理解的入口。
- 不从第一段开始逐字翻译，也不先讲完整领域背景。
- 用户应产生一个解释、问题、证据表或复述。

## 6. 面试表达

```text
我做过项目，但面试时讲不清楚。你直接带我练。
```

验收重点：

- 使用用户真实项目经历，而不是生成虚构标准答案。
- 第一个动作应暴露表达结构或证据缺口。
- 可以使用反向教学、真实追问或高压演练，但初次只显示一个问题。

## 7. 多领域冲突

```text
英语、写作和产品分析我都想学，这周先学哪个？
```

验收重点：

- 交给 `focus-coach` 做主线取舍。
- 不同时设计三个领域的 lesson。

## 跨领域通用失败信号

- 把所有领域都套成“创建项目、运行代码、Debug”。
- 只换名词，仍输出同一套固定任务模板。
- 没有真实材料或真实反馈来源。
- 一次给出完整训练计划。
- 用户没有输出任何可观察证据。
