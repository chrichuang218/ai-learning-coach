# learning-coach 冒烟验证 Prompt

合格输出必须有明确结论、诊断、下一步、练习和复盘记录；不能只给泛泛学习路线或资料清单。

## 1. 技术学习

```text
我想学 TypeScript / React / Agent，但收藏了很多教程还是不会用。
```

验收重点：

- 不能把 skill 限定成技术专用，但可以把技术作为当前案例处理。
- 先判断目标、背景、项目载体和最高杠杆动作。
- 给一个最小项目、源码切片或功能练习。
- 不能从语法路线、教程清单或完整课程表开始。

## 2. 英语口语

```text
我想练英语口语，能听懂一些内容，但自己说不出来。
```

验收重点：

- 构造真实说话场景或最小输出任务。
- 给可观察验收标准，例如录音长度、表达完整度或复述准确度。
- 失败反馈要包含录音问题、卡住句子或想表达但不会说的内容。

## 3. 备考

```text
我准备考试，做了很多题但分数没提升。
```

验收重点：

- 诊断是否缺少错题归因、题型识别、限时训练或复盘。
- 只安排一个 30-90 分钟复习闭环。
- 复盘记录必须有证据、真实卡点、影响和下一步。

## 4. 写作和短视频表达

```text
我想学写作和短视频表达，收藏很多案例，但自己写出来没重点。
```

验收重点：

- 绑定到一个真实输出物，例如标题、开头、脚本片段或改稿。
- 不给泛泛创作建议。
- 验收标准要能通过文本或口播稿判断。

## 5. 产品、运营或研究方法

```text
我想学产品分析和用户研究，但不知道怎么开始用到真实问题里。
```

验收重点：

- 构造一个最小研究问题或分析对象。
- 给出任务、验收标准和失败时反馈。
- 不把输出变成完整课程大纲。

## 6. 资源请求

```text
推荐几个学习 Python 的课程和书，我想系统学。
```

验收重点：

- 不能直接给长资源清单。
- 必须先把资源绑定到一个真实动作或目标阶段。
- 如果给资源，最多优先给一个主资源，并说明怎么用它完成当前闭环。

## 7. 学后复盘

```text
我刚照着教程完成了一个小项目，但离开教程就不会做。下一步怎么办？
```

验收重点：

- 不能直接安排下一章。
- 必须区分流畅感和长期能力。
- 应该安排不看答案的迁移验证，并要求用户带回具体证据。

## 8. 项目牵引

```text
我想学 React，但不想再从语法和教程开始。请基于一个项目来学。
```

验收重点：

- 必须先构造或选择一个最小项目动作。
- 练习要发生在项目、组件、功能切片或源码入口里。
- 只讲完成这个项目动作所需的知识。
- 失败反馈必须要求代码、报错、卡住步骤或解释。

## 通用失败信号

出现下面任一情况视为不合格：

- 主要内容是完整课程路线、书单或鼓励语。
- 技术学习没有项目动作、源码入口、功能切片或可运行实验。
- 没有可观察验收标准。
- 没有要求用户带回证据。
- 在背景缺失时完全停住，只问问题不推进。
- 把“看懂了、做完了、感觉会了”当成掌握证据。
