# focus-coach 冒烟验证 Prompt

合格输出必须有明确结论、诊断、行动、成功标准和下一步作业；不能输出泛泛鼓励、平均分配方案或不可验证建议。

## 1. 多目标收敛

```text
我同时想学英语、写作、AI、剪辑和产品分析，但每天时间很少。帮我制定学习计划。
```

验收重点：

- 不能平均分配多个方向。
- 必须指出最大约束和主要取舍。
- 应该输出一个本周期主线和一个最小验证动作。

## 2. 优先级排序

```text
我现在有三个任务：准备面试、做个人项目、学 React。每个都重要，我不知道先做哪个。
```

验收重点：

- 必须基于目标、截止时间、收益、风险和依赖关系排序。
- 不能说“三个都做一点”。
- 必须给 24-72 小时内的行动顺序。

## 3. 瓶颈诊断

```text
我最近一直很忙，但感觉没有进展。每天都在看资料、收藏工具、做计划。
```

验收重点：

- 必须区分表面忙碌和真实进展。
- 必须判断是否缺少真实输出、反馈或验收标准。
- 必须给一个能暴露瓶颈的动作。

## 4. 执行复盘

```text
我本周计划做完一个 demo，但实际只看了教程，没有写代码。帮我复盘。
```

验收重点：

- 必须根据计划和实际偏差复盘。
- 必须指出主要阻力和下一周期调整。
- 不能只说“下周继续努力”。

## 5. 学习使命定义

```text
我想学 AI Agent，但不知道学到什么程度才算够。先帮我定义学习使命。
```

验收重点：

- 必须判断学习目标服务什么项目、作品、职业或现实任务。
- 必须定义成功标准和当前不学的范围。
- 必须把结论转成 `learning-coach` 可接手的输入。

## 6. 信息不完整的决策

```text
我想换方向，但信息还不够。你先帮我判断现在该不该立刻换。
```

验收重点：

- 必须区分事实、假设、推测和不确定项。
- 必须指出立刻换和暂缓换的主要风险。
- 必须给一个低成本验证动作，而不是直接替用户拍板。

## 通用失败信号

出现下面任一情况视为不合格：

- 没有明确取舍。
- 同时安排多个高成本方向。
- 没有事实、假设或不确定项区分。
- 没有 24-72 小时内可执行动作。
- 用鼓励、口号或“你要更自律”代替诊断。
