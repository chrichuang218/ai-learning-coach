# focus-coach 冒烟验证 Prompt

`focus-coach` 是低频战略教练。它负责取舍和阶段判断，完成后应退出并把具体带学交给 `learning-coach`。

## 1. 多目标收敛

```text
我同时想学 TS、算法和多 Agent，但这个月只能重点投入一个。
```

验收重点：

- 先读取已有使命、背景和记录；如果它们已经表明现实目标，选择一条主线，而不是三个都安排。
- 如果没有任何现实目标，而且不同答案会改变选择，只问一个短问题或明确采用的临时假设，不能凭空断言用户要进入某个领域。
- 明确冻结或最低维护项。
- 给出一个周期、证据和重新评估条件。

## 2. 最大约束

```text
我最近每天都学，但两个月过去了还是不会独立做项目，问题在哪？
```

验收重点：

- 区分事实、假设和未知。
- 找一个最大约束，例如缺真实输出、反馈或路线错误。
- 设计最小验证，而不是直接归因于不自律。

## 3. 是否继续方向

```text
我学这个方向投入很多，但越来越怀疑它和我的工作目标无关，要不要停？
```

验收重点：

- 比较现实目标、沉没成本和机会成本。
- 给出继续、缩小、暂停或转向的明确当前判断。
- 设置可撤销条件。

## 4. 周期复盘

```text
帮我复盘这周，并决定下周唯一主线。
```

验收重点：

- 依据实际完成证据，不只依据感受。
- 输出一个周期承诺和复盘触发器。
- 不展开具体课程内容。

## 5. 学习使命

```text
我想学 TypeScript，但不知道学到什么程度才够。
```

验收重点：

- 连接现实项目、职业或作品结果。
- 定义“够用”的阶段能力和证据。
- 方向足够清楚后交给 `learning-coach`，不继续设计第一课。

## 6. 信息不完整

```text
我有两个工作机会，一个成长快但风险高，一个稳定但内容一般，怎么选？
```

验收重点：

- 先使用已有信息做当前判断。
- 只有答案会改变选择时才问少量问题。
- 明确风险、取舍和验证条件。

## 7. 不应触发的教学请求

```text
这段 await 为什么不会卡线程？
```

```text
开始学习 TS 下一课。
```

验收重点：

- 不进行战略分析。
- 交给 `learning-coach` 或直接进入相应技术帮助。

## 8. 基于进度做阶段取舍

```text
我的项目进度连续三周没动，但贡献日历一直有记录。帮我判断是继续、缩小范围还是转向。
```

验收重点：

- 读取 `PROGRESS.md`、使命、项目范围和近期 learning records，检查出勤是否真的产生掌握证据。
- 找到最大约束，不能把“每天有打卡”直接解释为路线有效。
- 给出继续、缩小或转向的明确判断、代价和复盘触发器。
- 不接管日常贡献日历和百分比更新，也不继续设计下一节课。

## 通用失败信号

- “两个都可以试试”。
- 同时安排多个高成本主线。
- 用长计划代替取舍。
- 把拖延一律解释成自律不足。
- 方向已清楚后仍反复使命诊断。
- 用连续学习天数代替能力、项目或现实结果判断。
- 给出战略后继续逐行教学。
