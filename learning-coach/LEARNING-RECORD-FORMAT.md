# 学习记录格式

学习记录放在 `learning-records/`。文件名使用递增编号：`0001-slug.md`、`0002-slug.md`。

学习记录不是流水账。它记录会影响后续教学判断的洞见、证据、卡点和目标变化。

## 模板

```md
# {短标题}

{1-3 句话：这次真正学会了什么、暴露了什么卡点、为什么会影响后续学习。}

## 证据

- {用户完成的输出、练习结果、回答、录音、草稿、代码、错题、解释或其他可观察证据}

## 真实卡点

- {用户具体卡在哪里。不要写“基础薄弱”这种泛话}

## 影响

- {这条记录会如何改变下一节课、后续复习或学习边界}

## 下一步

- {一个 30-90 分钟能完成的小闭环}
```

## 什么时候写

- 用户用证据证明自己掌握了一个非平凡概念。
- 用户暴露了后续会反复影响学习的卡点。
- 一个误解被纠正。
- 用户说明了已有背景，且这会改变后续教学方式。
- 学习使命或成功标准发生变化。

## 不要写什么

- 不记录“今天讲了什么”的流水账。
- 不把没有证据的“理解了”写成事实。
- 不重复 `GLOSSARY.md` 里已经压缩好的术语定义。
- 不把鼓励、感想和口号当学习记录。
