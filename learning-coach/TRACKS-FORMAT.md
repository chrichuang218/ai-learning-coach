# 学习轨道总览格式

`TRACKS.md` 用于多轨道学习工作区。它只做导航、状态和边界管理，不承载具体课程内容。

## 推荐模板

```markdown
# 学习轨道总览

这个工作区用于长期学习多个方向。每次只推进一个轨道的一个小闭环。

| Track | 状态 | 当前焦点 | 下一编号 |
| --- | --- | --- | --- |
| `<track>` | 活跃 / 暂停 / 预留 / 完成 | 当前最重要的小方向 | `<prefix>-0001-...` |

- 主线轨道：

## 分层规则

- 根目录：总学习目标、轨道导航、跨轨道偏好和协作规则。
- `tracks/<track>/`：该轨道的 `MISSION.md`、`STUDY-PLAN.md`、`RESOURCES.md`、`NOTES.md`。
- `lessons/<track>/`：该轨道单节小课。
- `exercises/<track>/`：该轨道练习、实验、题目、作品草稿或可验证输出。
- `reference/<track>/`：该轨道长期参考、框架卡、流程卡和对照表。
- `learning-records/<track>/`：该轨道证据型学习记录。
- `assets/`：跨轨道复用组件。
- `sources/`：外部读本、项目、课程、平台或资料源登记。

## 新增轨道流程

1. 给轨道起一个短的 dash-case 名称。
2. 在 `tracks/<track>/` 建立 `MISSION.md`、`STUDY-PLAN.md`、`RESOURCES.md`、`NOTES.md`。
3. 在 `lessons/`、`exercises/`、`reference/`、`learning-records/` 下建立同名子目录。
4. 如有外部资料源，先登记到 `sources/`。
5. 更新本文件的轨道清单、状态和当前焦点。

## 冻结规则

- 一个周期只保留一个主线轨道。
- 暂停轨道必须写清楚暂停原因和恢复条件。
- 不因为焦虑临时新增轨道；新增前先说明它服务什么目标。
```

## 使用原则

- `TRACKS.md` 不是计划堆放处。它只回答“有哪些轨道、现在先做哪个、边界是什么”。
- 每个轨道自己的使命、资源、学习计划和偏好写到 `tracks/<track>/`。
- 轨道状态要能帮助决策：活跃、暂停、预留、完成，不要写模糊状态。
- 新增轨道前先检查是否只是已有轨道的一个练习或资源。
