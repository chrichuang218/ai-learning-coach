# MISSION.md：学习使命格式

`MISSION.md` 是学习指南针：为什么学、学到什么程度、服务什么现实结果。它帮助 `focus-coach` 做战略取舍，也帮助 `learning-coach` 判断最近发展区。

## 推荐模板

```markdown
# Mission: {学习主题}

## Why

{1-3 句话，说明学会后要解决什么真实问题。}

## Success looks like

- {可观察的能力结果}
- {真实项目、作品、考试或工作结果}

## Completion criteria

- {什么独立行为或迁移证据表示当前目标已经达到}
- {哪些结果不属于本阶段完成要求}

## Constraints

- {基础、时间、环境、工具、风险和学习偏好}

## Out of scope

- {当前阶段明确不追的相邻主题}
```

## 规则

- 保持短；使命不是课程表。
- 多轨道工作区可以有多个轨道使命，但一个周期只保留一条高成本主线。
- 用户已经给出真实项目和明确用途时，不要为了补齐模板阻塞开始。
- 低风险缺失信息使用合理假设推进；只有不同答案会改变方向时才提问。
- 多目标冲突、是否值得继续或“学到什么程度才够”由 `focus-coach` 收敛。
- 现实目标变化时更新使命，不机械延续旧路线。
- `PROGRESS.md` 的目标能力里程碑从这里的成功标准与完成标准派生；使命本身不保存百分比。
