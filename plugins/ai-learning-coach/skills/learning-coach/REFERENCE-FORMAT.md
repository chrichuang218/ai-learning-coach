# Reference：稳定知识源格式

## 定位

`reference/*.md` 保存已经在真实学习中验证过、去个人化、可长期复用的稳定知识。

它既是本地可直接阅读的知识库，也是可选外部知识系统的 raw/source。用户没有 Obsidian、LLM Wiki、Notion 或其他工具时，reference 仍必须完整可用。

## 什么时候创建

适合创建：

- 一个概念经过对话、Run、Debug 或真实任务验证，心智模型已经稳定。
- 同一误区反复出现，值得形成长期纠错卡。
- 一个最小实验未来仍有复现价值。
- 多次 lesson 都需要引用同一知识单元。

不适合创建：

- 用户刚第一次听到概念。
- 只是一次路径、环境或截图差异。
- 结论仍不确定或只来自单次偶发现象。
- 只是聊天记录、课程流水账或个人情绪。

## 推荐模板

````markdown
# {稳定知识主题}

状态：稳定 | 待复验
来源：{真实项目、官方资料或实验}

## 要解决的问题

{未来复习时，用户想快速回答什么？}

## 一句话心智模型

> {...}

## 最小可复现例子

```{language}
...
```

预期输出或行为：

```text
...
```

## 机制解释

{用数据流、控制流、状态或时间线解释。}

## 真实项目映射

- `{path}`：`{symbol}` — {为什么这里体现该机制}

## 易错点

- 错误理解：{...}
  正确理解：{...}

## 已有背景对照

{例如 Java 对照；只在确实帮助理解时使用。}

## 复习问题

1. {...}

## 来源与可追踪性

- {...}
````

## 写作规则

- Markdown 是唯一默认主格式。
- 优先保存概念模型、可复现实验、预期输出、真实源码映射和易错点。
- 代码片段只保留理解机制所需的最小范围。
- 个人路径、临时截图和一次性环境差异不进入稳定正文。
- HTML 只有在交互演示确实增加理解时作为 Markdown 链接附件。
- 外部 Wiki 的派生页面必须保留来源，避免形成两份独立维护的真相。
