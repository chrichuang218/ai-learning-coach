# PROGRESS.md：学习进度与节奏格式

`PROGRESS.md` 是用户可见的学习仪表盘。它回答四个问题：目标能力走到哪里、约定的项目学习范围走到哪里、最近是否持续投入、按当前节奏还需要多久。

掌握百分比是派生状态；出勤部分只保存紧凑活动事实：

```text
MISSION.md
  -> 目标与完成标准

sources/<project>.md
  -> 项目学习范围与完成标准

learning-records/*.md
  -> 实际掌握证据与后续教学状态

PROGRESS.md
  -> 汇总掌握状态，并保存紧凑的出勤活动
```

出勤不等于掌握。打卡、阅读时长、lesson 数量和文件数量都不能直接提高目标进度或项目进度。

## 何时创建

满足以下条件时创建根目录 `PROGRESS.md`：

- 用户明确希望长期感知进度、连续学习、成就或剩余时间。
- 工作区已经有一个可描述的使命，以及至少一项目标能力或项目学习范围。
- 学习会跨会话持续，仪表盘能改善后续选择和动力。

一次短答疑不需要创建。多轨道工作区默认只展开当前高成本主线；冻结轨道可以显示一句状态，但不同时维护多套详细仪表盘。

## 掌握状态

每个里程碑只能使用以下状态：

| 状态 | 数值 | 证据含义 |
| --- | ---: | --- |
| 未开始 | 0% | 尚未形成可用证据 |
| 接触 | 25% | 看过或在支持下识别过，但不能稳定解释 |
| 需提示 | 50% | 能解释或完成，但仍需要明显提示、模板或共同操作 |
| 独立 | 75% | 能在当前项目或同类任务中独立解释、预测、操作或修改 |
| 迁移验证 | 100% | 能在新位置、新任务或变化后的条件中独立使用并验证结果 |

状态依据最新有效证据更新。不要因为时间过去自动降级；只有复习或新任务暴露真实能力变化时才调整。

## 进度计算

1. 目标能力里程碑的权重合计必须为 `100%`。
2. 项目范围里程碑的权重合计必须为 `100%`。
3. 原始进度按 `sum(权重 x 掌握状态) / 100` 计算。
4. 用户可见进度取最接近的 `5%`，避免制造虚假精度。
5. 只有 `learning-records/` 中的解释、预测、Run、Debug、修改、作品、反馈或迁移证据可以改变掌握状态。
6. 同一证据可以同时推动一个目标能力和一个项目范围里程碑，但不能在同一张表里重复计权。

“学完项目”表示完成 `sources/<project>.md` 约定的学习范围，并通过独立或迁移验收；不表示读完仓库每个文件。目标或范围变化时重新建立基线，并在 `PROGRESS.md` 说明原因，不把范围变化伪装成能力退步。

## 时间估算协议

- 同时给出剩余有效学习小时和按当前节奏换算的周区间。
- 周区间使用 `最少剩余小时 / 每周最多小时` 到 `最多剩余小时 / 每周最少小时` 计算，并做合理取整。
- ETA 必须写出学习范围、每周节奏、外部依赖和置信度；不要只给一个完成日期。
- 不从进度百分比线性反推时间。根据未完成里程碑的难度、独立性要求和已观察节奏估算。
- 少于 3 个有效学习日、没有可靠时长或尚无里程碑移动时，置信度保持低并标记“待校准”。
- 记录了至少 3 个有效学习日且出现一次里程碑升级后，可以按真实节奏重新估算；稳定跨越两个以上学习周后才考虑高置信度。
- `有效学习时间` 只有用户明确提供、工具可靠记录或会话时间可准确判断时才写入；不能猜测精确分钟数。

## 贡献与连续学习

- 出现掌握证据、误区纠正或后续教学状态变化时，先写 learning record，并在其中写贡献等级。
- 只有低强度出勤、没有内容值得进入 learning record 时，直接在 `PROGRESS.md` 的“最近活动”和贡献日历记录，不创建伪证据记录。
- 同一天有多条记录时，日历只显示最高等级，不累加制造活跃度。
- `░` 可以增加出勤日和连续学习天数，但不会自动提高掌握进度。
- `▒`、`▓`、`█` 也只有在证据足以改变某个里程碑状态时才提高进度。
- 只从启用进度功能当天开始记录；不要根据“以前学过很多”批量补满历史日历。
- 空白日只表示没有有效学习记录，不扣分，也不降低已经证明的能力。
- 默认保留最近 12 个 ISO 周；更早记录可以按月汇总，避免文件无限增长。

## 更新顺序

每次产生有效学习活动后：

1. 先判断是否出现掌握证据、误区纠正或教学状态变化；有则创建或更新 `learning-records/`，没有则保持 record 不变。
2. 在“最近活动”写一行当天最高强度和真实动作；有关联 record 时添加链接。
3. 判断新证据是否改变一个目标能力或项目范围里程碑；没有改变就保持百分比不动。
4. 更新当天贡献格、连续学习、本周与累计有效学习日。
5. 重新计算两个加权进度和进度条。
6. 在达到校准条件时更新 ETA；否则保留原区间并说明新证据。
7. 只保留最近 2-5 个成就，并把下一检查点写成一个可观察证据。

`PROGRESS.md` 不能取代 learning record，也不要复制每轮完整学习叙述。它只展示当前状态、节奏和方向，并为没有掌握证据的低强度出勤保存一行紧凑事实。
