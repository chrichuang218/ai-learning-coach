## 三种学习结果

复杂技能通常包含三个层次：

- **知识**：事实、概念、术语、模型和规则。
- **技能**：能够执行、判断、调试、表达或解决问题。
- **智慧**：在真实限制、他人反馈和不完整信息中做出合适取舍。

不同领域比例不同：

- 编程和口语偏技能。
- 考试包含知识、识别和执行速度。
- 写作、产品和研究需要大量真实反馈与判断。

不要只因为用户能复述知识，就宣称他拥有技能或智慧。

## 流畅感不是掌握

看懂解释、跟着做完、熟悉答案会产生流畅感。流畅感有用，但很容易被误认为长期能力。

最低证据梯度：

```text
看过
-> 能识别
-> 能解释
-> 能预测
-> 能独立操作
-> 能迁移到新场景
-> 能在真实限制下稳定使用
```

根据主题风险选择最低充分证据，不要求每个概念都走完整梯度。

## 主动回忆

主动回忆要求用户在不看答案时重新产生关键内容。

适合：

- 复述执行时间线。
- 写出解题步骤。
- 说明一段代码为什么安全。
- 回忆真实任务中的动作顺序。

私人教练的做法：只问当前最关键的一件事，不一次发整套测验。

## 间隔复习

刚学完时反复回答同一问题，通常只能证明短期可用。重要概念应在后续真实场景自然再次出现。

使用方式：

- 在 learning record 中保存下一次值得复验的边界。
- 当相同机制在新源码、题目或任务出现时进行短回忆。
- 用户已稳定掌握时停止重复，不为了“间隔”机械出题。

## 交错练习

当基础稳定后，把容易混淆的相邻机制放在一起判断：

- `await` 当前 Promise 与未等待的后台 Promise。
- `type`、`interface` 和 runtime validation。
- 相近题型或写作结构。
- 不同 provider 的消息 DTO 与统一内部模型。

目标是练习“何时使用哪一个”，不是增加题量。

## 必要难度

有效练习应该有一点吃力，但仍能开始。

提高难度的方法：

- 少给一个提示。
- 先预测再运行。
- 改变一个条件。
- 延迟查看答案。
- 要求解释边界。

降低难度的方法：

- 缩小到一个值或一行。
- 给匿名回调命名。
- 展开时间线或状态。
- 提供部分结构。
- 现场演示一个工具操作。

不要把困惑、猜谜和信息过载误称为“必要难度”。

## 反馈速度

技能形成需要紧反馈：

- 编程：编译、测试、Run、Debug、日志和真实用户行为。
- 口语：录音、转写、真实对话和听众理解。
- 写作：读者是否抓住重点、编辑反馈和修改前后对比。
- 考试：限时得分、错因和重复错误模式。
- 产品：用户访谈、行为数据、实验和实际决策。
- 研究：证据质量、可复现性、同行质疑和预测结果。

教练应优先选择反馈最快、成本可控、最能暴露当前卡点的真实动作。

## 真实世界反馈

有些能力无法只在对话中证明，例如沟通、写作影响力、产品判断、研究判断和真实工程协作。

基础动作建立后，应逐步接入：

- 用户、读者、老师、同伴、面试官、客户或社区。
- 真实会议、公开作品、模拟考试、真实对话或生产项目。
- 可观察指标：错误率、完成时间、理解率、通过率、回复率或实际结果。

尊重隐私和风险约束。用户不愿公开时，先设计低风险替代反馈。

## 跨领域真实锚点

| 领域 | 真实锚点 | 可观察证据 |
| --- | --- | --- |
| 编程/源码 | 函数、调用链、Bug、功能、测试 | 预测、运行、Debug、修改、测试 |
| 语言 | 一次表达、录音、对话 | 是否表达完整、卡点、听者理解 |
| 考试 | 代表题、错题、限时组卷 | 得分、耗时、错因、迁移题 |
| 写作 | 标题、段落、完整草稿、改稿 | 读者理解、结构、修改效果 |
| 产品/职业 | 决策、会议、用户问题、交付物 | 实际反馈、采用情况、指标 |
| 研究 | 问题、证据表、实验、论文段落 | 可复现性、证据强度、预测 |

## 资源与事实可靠性

资源服务当前动作，不替代动作。

- 优先一个主资源，不给收藏清单。
- 易变化的 API、考试要求和官方规则优先查一手来源。
- 法律、医学、金融等高风险知识需要明确来源和不确定性。
- 模型记忆不能替代当前官方文档和真实运行证据。
- 未使用、未验证的资料不进入稳定 reference。

## 工作区如何支持长期能力

- `LEARNER-BACKGROUND.md`：保存已经证明的能力和边界。
- `learning-records/`：保存一次真实掌握证据和下一复验点。
- `reference/`：保存稳定知识，不保存个人流水账。
- `STUDY-PLAN.md`：保存短主线，安排未来自然出现的迁移机会。
- lesson brief：只在跨时间恢复或复用设计时保存教学现场。

不要为每个学习科学原则创建额外文件。让它们体现在下一次真实动作里。
