# Learning Record：个人掌握证据格式

学习记录保存“这个用户已经用什么证据证明了什么”，让教练下次不用重新猜测起点。

它不是 lesson brief、课程总结或学习日记。

## 推荐模板

```markdown
# {短标题}

日期：{YYYY-MM-DD}
真实锚点：`{项目路径、题目、作品或任务}`
贡献等级：{`░` / `▒` / `▓` / `█`}
有效学习时间：{可选；只记录已知值，不猜测}
推进里程碑：{目标能力或项目范围中的一个或多个名称；没有升级也可写“无”}

## 本轮结论

{一句话说明本轮真正建立了什么能力。}

## 掌握证据

- {用户做出的解释、预测、运行、Debug、修改、作品或迁移行为。}

## 真实卡点与纠正

- 原模型：{用户原来怎样理解。}
- 纠正：{什么证据改变了理解。}

## 后续可默认

- {下一次教学可以不再从定义重讲的内容。}

## 能力边界

- {仍未证明或需要在新场景迁移的部分。}

## 下一处真实入口

- `{path}`：`{symbol / task}`
```

## 什么时候写

- 用户通过非平凡行为证明掌握。
- 一个真实误解被证据纠正。
- Run、Debug 或真实反馈改变了下一步判断。
- 学习使命、能力边界或有效教学方式发生变化。
- 一次有效复习没有提高掌握状态，但明确验证了保留程度或改变了下一入口；这类记录保持简短。

只有阅读或出勤、没有掌握证据也没有改变下一次教学判断时，不创建 learning record；只在 `PROGRESS.md` 的最近活动和贡献日历记录一行。

## 贡献等级

- `░`：回忆、复习或阅读；可以计入有效学习日，但不自动提高掌握进度。
- `▒`：主动解释、预测或源码追踪。
- `▓`：Run、Debug、修改或产生真实输出。
- `█`：里程碑、迁移验证或独立完成。

同一天有多条记录时，`PROGRESS.md` 的贡献日历只取最高等级。贡献等级描述学习活动强度，不代替“掌握证据”部分。

## 不要写什么

- “今天学了某某”式流水账。
- 只因用户说“懂了”就标记掌握。
- lesson 的完整教学计划。
- 尚未稳定的通用知识文章。
- 临时截图、个人路径差异或偶发输出。
- 为了维持连续天数而编造学习活动、时长或掌握提升。
