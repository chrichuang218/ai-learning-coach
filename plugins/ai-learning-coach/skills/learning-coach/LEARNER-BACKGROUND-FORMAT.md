# LEARNER-BACKGROUND.md：学习者背景格式

学习者背景保存后续教学可以默认使用的个人上下文：已有经验、经过证据证明的能力、当前边界和有效教学方式。

它不是课程目录，也不要求用户重新阅读。学习目标、现实用途、完成证据和当前主线属于 `MISSION.md`；本文件只保存相对稳定的学习者背景。

## 推荐模板

```markdown
# {主题} 学习者背景

更新日期：{YYYY-MM-DD，可选}

## 已确认背景

- {用户明确陈述的职业、语言、项目或领域经验}

## 可迁移经验

- {旧经验} — 可能帮助：{新领域中的对应能力}

## 临时假设

- {尚未确认的能力或经验} — 依据：{为什么暂时这样判断}；验证：{如何在真实任务中确认}

## 已验证能力

- {能力} — 证据：{解释、预测、运行、Debug、作品、考试或真实反馈}

## 已建立的心智模型

- {...}

## 有效教学方式

- {...}

## 不适合的教学方式

- {...}

## 当前能力边界

- {见过但尚未证明能独立使用的内容}

## 下一处真实入口

- `{path / task}`：`{symbol / focus}`
```

## 写入规则

- 只保存会改变后续教学判断的稳定背景。
- 先读取对话、项目和已有记录；缺失信息确实会改变学习起点时，才主动询问一个短问题。
- 不发送背景问卷。用户无法回答时，可以记录明确标注的临时假设，并通过真实任务继续校准。
- 用户明确陈述的信息、教练推断的临时假设和证据验证的能力必须分开记录。
- “学过”不等于“已验证能力”。
- 可以汇总多个 learning record，但不要复制完整学习流水账。
- 明确区分已经掌握、需要提示、尚未验证三种状态。
- 用户最新明确偏好优先于旧背景；发生变化时直接更新。
- 学习目标和完成定义写入 `MISSION.md`；不要把不断变化的短期目标塞进背景文件。
- 个人背景不应被压缩成去个人化 reference，两者职责不同。
